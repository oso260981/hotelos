Fase 1: Pasos Lógicos para Generar el Reporte
El reporte no se genera "mágicamente"; requiere un flujo de trabajo en tu backend de Node.js (Electron) antes de enviarlo a la impresora térmica.

Identificar el Turno a Cerrar: Obtener el id del turno que tiene estado = 'ABIERTO'.

Ejecutar Consultas de Sumarización (Queries): Lanzar las consultas SQL (ver Fase 2) para calcular ocupación, ingresos por hospedaje, ingresos por room service (pagado en efectivo) y egresos.

Calcular el Arqueo de Caja:

Efectivo_Calculado = Fondo_Fijo_Inicial + Ingresos_Hospedaje_Efectivo + Ingresos_RS_Efectivo - Egresos_Efectivo.

Solicitar Efectivo Real: Pedir al recepcionista que ingrese cuánto dinero físico tiene.

Actualizar Base de Datos (Cierre): Guardar hora_cierre, efectivo_real, diferencia_arqueo y cambiar estado a 'CERRADO'.

Formatear e Imprimir (ESC/POS): Tomar todos los datos calculados, armar el diseño del ticket y enviarlo a la impresora térmica.

Fase 2: Queries SQL (El Motor del Reporte)
Estas son las consultas críticas que debes ejecutar en SQLite para obtener los números precisos del turno.

1. Resumen de Ocupación del Turno
SQL
SELECT 
    COUNT(CASE WHEN estado_registro = 'CHECKIN' THEN 1 END) as total_checkins,
    COUNT(CASE WHEN estado_registro = 'CHECKOUT' THEN 1 END) as total_checkouts,
    SUM(adultos + niños) as total_personas_registradas
FROM registros
WHERE turno_id = ? AND incluir_en_reporte = 1;
2. Ingresos por Hospedaje desglosados (Crucial para Arqueo)
SQL
SELECT 
    forma_pago_id, -- Necesitas mapear esto (1=Efectivo, 2=Tarjeta, etc.)
    SUM(total) as total_ingresado,
    SUM(iva) as total_iva,
    SUM(ish) as total_ish
FROM registros
WHERE turno_id = ? AND incluir_en_reporte = 1 AND estado_registro != 'CANCELADO'
GROUP BY forma_pago_id;
(Nota: Para el arqueo de efectivo, solo usas la suma donde forma_pago_id sea el de 'Efectivo').

3. Ingresos por Room Service (Detalle de Línea)
(Asumiendo que observaciones indica si se pagó en efectivo o se cargó a la cuenta)

SQL
SELECT 
    SUM(subtotal) as total_rs_efectivo
FROM registro_room_service
WHERE turno_id = ? AND estado_cargo = 'ENTREGADO' AND observaciones LIKE '%PAGO EFECTIVO%';

Contexto y Requerimientos Técnicos:

Tablas de Input: Tengo las tablas historial_turnos (para saber el fondo inicial y estado), registros (para hospedaje) y registro_room_service (para cargos extra). Las dos últimas tienen el campo turno_id perfectamente vinculado.

Ocupación: Total Check-ins y Check-outs del turno.

Arqueo: Calcular efectivo_esperado (Fondo + Ingresos_Efectivo_Hospedaje + RS_Efectivo - Egresos).
